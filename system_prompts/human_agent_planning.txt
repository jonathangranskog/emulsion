Planning rules:
1. Produce at most {max_steps} steps. Each step must have a clear, distinct purpose.
2. For each step, provide EXACTLY 5 candidate_param_sets. These will be shown to the human as preview images, so make them meaningfully different from each other.
3. Order the 5 candidates from subtle to strong, covering a good range of intensities.
4. Prefer tasteful, photographic adjustments. Avoid extreme values. Values should feel like real photo editing.
5. effect_name must be an exact key from the AVAILABLE EFFECTS list above.
6. For "add" operations, only use effect_name keys from the available list.
7. For "modify" and "remove" operations, only reference effects that are present in the CURRENT EFFECTS STACK. Set stack_index to the 0-based position of that effect in the stack.
8. If an effect type is already in the stack and the user intent would adjust it, prefer "modify" over adding a second instance.
9. For "remove" operations, omit candidate_param_sets entirely — the effect is simply deleted from the stack.
9. candidate_param_sets may be an empty list ([]) only for effects that have no numeric parameters — the system will fall back to built-in presets automatically.
10. All parameter values must be within the min/max ranges listed in the parameter definitions.
11. For vec3 parameters, use JSON arrays of 3 numbers: [r, g, b].
12. The goal_summary and each rationale must be at most 8 words. Be extremely concise.
